浅析递归中的return

很多人把递归比喻成

从前有座山，山上有座庙，庙里有个老和尚，老和尚对小和尚说：
从前有座山，山上有座庙，庙里有个老和尚，老和尚对小和尚说：
……

我觉得这是不严谨的，因为没有递归的终止条件，这样会内存溢出的；

对于 return true，return false这种，可以参考任何一个讲递归的博客，应该都有讲；我们这里讲稍微复杂一点的，return 后面还是递归函数，也就是说什么时候递归函数前面要加return；什么时候不加return；

举个例子🥔

我们假设在十天之内你一定会中彩票，彩票的金额有10块，和100块两种，并且在这十天，你一定会中到100块，这就是你的目标，当你中到100块时，你就不买了，当然也有可能中10块，中10块停一天再买；以上是游戏的规则。

我们这里讨论的是第5行的买彩票前要不要加上return，答案是要的

如果不加，我们假设执行完第5行的买彩票，中了100块，然后返回true，返回到第5行一个true，没有return，也就是相当于没有返回true，中100块等于白中了，会继续执行第6行的买彩票。此时逻辑已经错误了。

如果加了return，返回到第5行之后，相当于 return true，继续返回上一层，此时逻辑才是正确的；

什么时候递归函数前面加return呢？

如果前面是满足一定的条件才执行这个递归函数呢，那么需要加上return。
