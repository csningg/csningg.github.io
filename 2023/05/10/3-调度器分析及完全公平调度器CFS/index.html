3.调度器分析及完全公平调度器CFS

发表于 2023-05-10 分类于 Linux内核源码分析 阅读次数：
本文字数： 6.2k 阅读时长 ≈ 6 分钟

本篇文章主要进程调度的相关内核代码及公平调度器CFS是怎么运行的

调度器分析

调度器及其功能

内核中用来安排进程执行的模块称为调度器（scheduler），它可以切换进程状态（process state）。例如执行、可中断睡眠、不可中断睡眠、退出、暂停等。

调度类sched_class结构体与调度类

Linux中有5中调度类，每中调度类有不同的调度策略。比如我们经常听说的时间片轮转算法，先进先出调度算法等。这些都是调度策略。每个进程都会对应一个调度策略，一个调度策略对应一个调度类（而每个调度类有多重调度策略）

那么这五种调度类的结构体是一样的，定义在 kernel/sched/sched.h的sched_class中。

比较重要的成员函数有：

enqueue_task,dequeue_task,yeild_task,check_preempt_curr,pick_next_task,put_prev_task,set_curr_task,task_tick

以上函数是功能在代码中都有解析

调度类

Linux的调度类有5中

stop_sched_class：最高优先级的调度类型，属于实时调度类型的一种，会抢占其他所有的进程。

dl_sched_class：deadline调度类，实时调度中较高级别的调度类型，一般之后在系统紧急情况下会调用。使用红黑树，有三个参数（运行周期，运行时间，截止时间）作为调度依据，用不到优先级，所以优先级设置为-1,

rt_sched_class：实时调度类型，软实时调度器，希望立刻运行，优先级为0-99,

fair_sched_class：现在较高版本的Linux上也就是CFS(Completely Fair Scheduler)，Linux上面主要的调度方式，引入虚拟时间的方式，优先级为100-139。

idle_sched_class：优先级最低，在系统空闲时才会进入该调度类型调度，属于最低优先级的。

调度策略

并不是每一种调度类都有调度策略，总共有6中调度策略，如下所示

其中 SCHED_NORMAL,SCHED_BATCH,SCHED_IDLE直接别映射到fair_sched_class;

SCHED_RR,SCHED_FIFO与rt_sched_class进行关联。

SCHED_NORMAL:用于普通进程，通过CFS实现

SCHED_BATCH:相当于SCHED_NORMAL的分化版本，采用分时侧策略，根据动态优先级，分配CPU运行所需要的资源。用于非交互处理器消耗类型

SCHED_IDLE ： 最低优先级，只有空闲的时候才会执行，在系统负载很低的时候使用CFS

SCHED_FIFO:先进先出调度算法，实时调度策略，根据优先级任务先到先服务，高优先级任务可以抢占低优先级任务

SCHED_RR:轮转调度算法（实时调度策略）

SCHED_DEADLINE:新支持的实时进程调度策略，针对突发性机刷。

优先级

task_struct结构体中采用三个成员表示进程的优先级：prio和normal_prio表示动态优先级,static_prio表示进程的静态优先级。

内核将任务优先级划分，实时优先级范围是0到MAX_RT_PRIO-1（即99），而普通进程的静态优先级范围是从MAX_RT_PRIO到MAX_PRIO-1（即100到139）。

进程分类：

实时进程（Real_Time Process）:优先级更高，需要立刻被执行的进程。

普通进程（Normal Process）：低优先级，更长执行的进程，

进程的优先级是用0-139表示，数字越小，优先级越高，其中0-99留给实时进程，100-129留给普通进程。

完全公平调度器

CFS原理

CFS是一种新的模型，其给cfs_rq（csf的run queue）的每个进程安排一个虚拟时钟，vruntime，如果一个进程得以执行，随着时间的增长，其vruntime将不断增大，没有得到执行的vruntime不变。

而调度器总是选择vruntime跑得最慢的那个进程来执行。这就是所谓的“完全公平”。为了区别不同优先级的进程，优先级高的进程vruntime增长得慢，以至于它可能得到更多的运行机会。

CFS的基本涉及思路

为了维护完全公平调度算法，定义另个时间。

  1. 实际运行时间 ：分配给进程的运行时间 = 调度周期 * 进程权重 / 所有进程权重之和
    1. 调度周期，是指所有进程运行一遍所需要的时间
    2. 进程权重，根据进程的重要性，给每个进程分配不同的权重
  2. 虚拟运行时间：vruntime = 实际运行时间 1024 / 进程权重 = (调度周期 进程权重 / 所有进程总权重) 1024 / 进程权重 = 调度周期 1024 / 所有进程总权重

所以，虽然每个进程的权重不同，但是他们的虚拟运行时间都是一样的，与权重无关，既然所有进程的vruntime增长速度一样的，与权重无关，那么vruntime增长速度宏观上看应该是同时推进的。那么就可以用这个vruntime来选择运行的进程，谁的vruntime值较小就说明它以前占用cpu的时间较短，受到了“不公平”对待，因此下一个运行进程就是它。这样既能公平选择进程，又能保证高优先级进程获得较多的运行时间。这就是CFS的主要思想了。

CFS的思想就是让每个调度实体（没有组调度的情形下就是进程，以后就说进程了）的vruntime互相追赶，而每个调度实体的vruntime增加速度不同，权重越大的增加的越慢，这样就能获得更多的cpu执行时间。

可以看到进程在一个调度周期内的vruntime值大小与进程权重无关，所有进程的vruntime值在一个周期内增长是一致的。vruntime值较小的进程，说明它以前占用cpu的时间较短，受到了不公平对待，因此选择作为下一次运行的进程。
这样既能公平选择进程，又能保证高优先级进程获得较多运行时间，就是cfs的主要思想了。其可以简单概括为：机会平等、时间差异。
